# 📦 Delivery Delay Analysis & Performance Optimization

This project analyzes delivery performance to identify why deliveries are getting delayed and how to improve operational efficiency.

## 🔍 Problem Statement

Why are deliveries getting delayed and how can we improve delivery performance?

---

## 🔄 Workflow

1. **Data Collection**

   * Used a structured dataset with delivery, logistics, and operational fields

2. **Data Cleaning & Preparation**

   * Handled missing values (e.g., Delay Reason → “Unknown”)
   * Created calculated columns like Delay Days and Delay Status

3. **Data Modeling**

   * Built relationships and structured data for analysis
   * Created DAX measures for KPIs

4. **Analysis (WHAT – WHERE – WHY)**

   * WHAT: Identified delay percentage and average delay
   * WHERE: Analyzed delays by region, carrier, and warehouse
   * WHY: Investigated root causes like processing time, dispatch time, and delay reasons

5. **Visualization**

   * Designed a multi-page Power BI dashboard:

     * Overview
     * Root Cause & Business Impact

6. **Insights & Recommendations**

   * Derived key insights and provided business actions to improve delivery performance

---

## 📊 Key Insights

* 62% of deliveries are delayed, indicating a consistency issue
* Average delay is ~1.5 days (frequent small delays)
* Processing and dispatch time are main causes
* Carrier performance impacts delays
* Distance has minimal effect
* Data quality issue: many “Unknown” delay reasons

---

## 🚀 Business Recommendations

* Optimize warehouse processing and dispatch time
* Replace or improve underperforming carriers
* Improve delay tracking and data quality
* Focus on medium-priority and consumer orders
